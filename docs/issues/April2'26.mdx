# Boilerplate Review and Hardening Plan

## Summary
- The repo has a solid feature spread, but the current baseline is not production-safe yet. The biggest gaps are privilege enforcement, cross-origin/auth trust boundaries, Electron hardening, and release drift between the live repo and the scaffold template.
- I validated this with code inspection plus `pnpm --filter web test`, `pnpm --filter web typecheck`, and `pnpm --filter web lint`. `typecheck` passed, `lint` passed with many warnings, and `test` failed in 6 places, which is a release-readiness signal.

## Key Changes
- Introduce a real server auth layer for tRPC: `publicProcedure`, `protectedProcedure`, and `adminProcedure`; move session lookup into context; require server-side role checks for every admin/CMS mutation.
- Lock down cross-origin behavior: remove blanket `/api/trpc` and `/api/scaffold` public exemptions, stop reflecting arbitrary origins, add CSRF protection for state-changing routes, and use `SameSite=Lax` unless a specific cross-site flow truly needs otherwise.
- Harden desktop: re-enable Electron sandboxing/context isolation, unify on one deep-link scheme, and replace `null`/`file://` auth trust with a narrower desktop transport.
- Harden uploads: verify magic bytes, re-encode images server-side, sanitize keys, and prefer private buckets plus signed URLs where practical.
- Collapse template duplication: generate the distributable scaffold from one source of truth instead of maintaining a second repo copy.
- Add a typed env schema with explicit `server` vs `public` variables and stop using `NEXT_PUBLIC_*` to drive privileged server behavior.
- Add missing data integrity rules: FK from `Transaction.userId` to `User`, indexes for common queries, and stricter payment/webhook invariants.

## Test Plan
- Add regression tests proving anonymous and non-admin callers get `401/403` for CMS mutations.
- Add CSRF/CORS tests for `/api/scaffold` and auth routes, including hostile-origin requests.
- Add Electron auth smoke tests around the chosen deep-link scheme and session restoration.
- Add upload tests for spoofed MIME types and oversized/invalid image payloads.
- Make CI block merges on test, typecheck, and lint warning budgets; add a template-sync check so root and scaffold can’t silently drift.

## Assumptions
- I’m assuming Better Auth admin endpoints still enforce admin role server-side; if they do not, the severity of the exposed admin clients increases further.
- I’m assuming `templates/saas-boilerplate` is the shipped scaffold source, not just an internal example.
- I’m treating this as a production-oriented boilerplate review, so the recommendations bias toward secure defaults over local-dev convenience.
